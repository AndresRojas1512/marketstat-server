# Приложение для анализа зарплат на российском рынке труда

## Название проекта: MarketStat

## Цель
Целью проекта является создание инструмента для проведения многомерного анализа данных о заработных платах на российском рынке труда. Система предоставляет пользователям возможность формировать аналитические отчеты (бенчмарки) на основе гибкой фильтрации по набору измерений, таких как география, отрасль, должность и квалификация.

## Решаемая проблема
Проект предоставляет инструмент для HR-специалистов, аналитиков и обладателей предприятий. В условиях постоянно меняющегося рынка труда он позволяет получать аналитику по зарплатным трендам.

## Функциональные требования
Система должна обеспечивать выполнение следующего функционала:
- аутентификацию и авторизацию пользователей на основе ролей;
- предоставление программного интерфейса для вызова аналитических функций, таких как расчет перцентилей, медианы и среднего значения зарплат, удовлетворяющих указанным фильтрам.
- возможность выполнения фильтрации данных по всем ключевым измерениям;
- возможность сохранения результатов анализа для зарегестрированных пользователей;
- предоставление публично доступных (гостевых) функций, предоставляющих агрегатную информацию по предустановленным параметрам.

## Описание ролей

- **Аналитик**: Аутентифицированный пользователь, которому предоставлен доступ к полному набору инструментов для проведения детального анализа. Основная функция роли — формирование аналитических отчетов по настраиваемым параметрам
- **Гостевой (анонимный) пользователь** - любой неаутентифицированный пользователь. Пользователь с этой ролью имеет доступ только к ограниченному набору функций и параметров фильтрации зарплат.
- **Администратор** - администратор прилржения, имеющий доступ к всем объектам базы данных, учетным записям пользователей.

## Описание основных пользовательских сценариев
### 1. Гостевой (анонимный) пользователь
1. Инициация запроса: Пользователь обращается к одной из публичных аналитических функций системы.
2. Задание параметров: Пользователь указывает обязательные и опциональные параметры запроса, такие как отрасль и географический регион.
3. Задание параметров: Пользователь указывает обязательные и опциональные параметры запроса, такие как отрасль и географический регион.
4. Получение результата: Пользователь получает структурированный набор данных, содержащий обобщенную статистику, например, перечень должностей с указанием среднего уровня заработной платы. Сохранение результатов и доступ к расширенной аналитике недоступны.
---
### 2. Аналитик
1. Аутентификация: Пользователь проходит процедуру аутентификации в системе для получения авторизованного доступа к расширенному функционалу.
2. Конфигурация отчета: Пользователь формирует аналитический запрос, задавая комбинацию критериев фильтрации из доступных измерений: профессиональная область, должность, уровень квалификации, географическое положение (город, область, федеральный округ), временной период.
3. Выполнение анализа: Система обрабатывает запрос, выполняя комплексное вычисление по отфильтрованному набору данных. Результатом является бенчмарк-отчет, состоящий из трех аналитических блоков:
- статистическая сводка - метрики (среднее, медиана, перцентили).
- распределение данных - гистограмма.
- временной ряд - данные для анализа динамики показателей во времени.
4. Сохранение отчета: Пользователь инициирует операцию сохранения полученного отчета. Система фиксирует как параметры запроса, так и его результат в персональной истории пользователя для последующего доступа.
---
### 3. Администратор
1. Аутентификация - пользователь проходит аутентификацию с учетными данными, обладающими правами на выполнение операций по управлению данными.
2. Управление данными системы (добавление нового города или работодателя).
